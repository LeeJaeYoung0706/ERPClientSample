# ERPClientSample

본 프로젝트는 **ERP 연동 구조 설계 예제**를 목적으로 작성된 **샘플 Client** 입니다.  
실제 운영 환경이나 고객사 시스템에서 사용된 코드가 아니며,  
외부 시스템 연동 시 고려해야 할 **구조 설계 및 확장성 패턴**을 설명하기 위한 예제입니다.
샘플 페이지로 변경하면서 매개변수 주입이 잘못되었을 가능성이 존재합니다.
--

## 📌 프로젝트 목적

- ERP 시스템(Sample ERP) 연동 시 필요한  
  **Client 구조 설계 방식**을 예제로 정리
- 단기간 개발 환경에서도  
  **확장성과 유지보수를 고려한 코드 작성 방법 제시**
- 신규 API 연동 시 **공통화된 구조를 통해 빠르게 확장 가능한 기반 제공**

---

## ⚠️ 안내 (중요)

본 저장소에 포함된 모든 코드는 다음 기준을 따릅니다.

- 실제 고객사명, 회사명, 시스템 정보 **미포함**
- 실 서비스 URL, IP, 인증 정보, 계정 정보 **미포함**
- 실제 ERP 내부 로직 및 영업 비밀 **미포함**
- 연동 형식, 데이터 구조는 **일반화 또는 예시용으로 재구성**

👉 **실제 업무 코드가 아닌, 구조 및 설계 예제용 코드**입니다.

---

## 🛠️ 주요 설계 포인트

### 1️⃣ ERP 연동 Client 구조 표준화
- ERP API Endpoint를 Enum으로 관리하여
  - 호출 위치 명확화
  - URL 오타 및 잘못된 호출 방지
- 공통 요청/응답 처리 로직을 분리하여
  - 중복 코드 최소화
  - 신규 API 추가 시 설정 중심 확장 가능

### 2️⃣ DTO 기반 계약(Contract) 설계
- 외부 시스템 응답 스펙을 DTO로 정의
- 외부 응답 필드 변경에 대비한 유연한 역직렬화 설정
- 연동 계층과 비즈니스 계층의 책임 분리

### 3️⃣ 예외 및 응답 검증 전략
- 외부 API 비정상 응답에 대한 공통 예외 처리
- 상위 서비스에서 일관된 방식으로 오류 처리 가능하도록 설계
- 운영 환경에서 원인 추적이 가능하도록 구조화

## 💡 활용 예시

- 외부 API 연동(Client) 공통 프레임 구성
- 단기간 프로젝트에서 빠르게 적용 가능한 표준 템플릿
